# Milestone 2.5 — CI Pipeline + App Platform Deployment

## Overview

Set up GitHub Actions CI (lint + test on every PR) and DigitalOcean App Platform deployment (auto-deploy from main). After this milestone, every merge to main auto-deploys to production.

## CI Pipeline

**File**: `.github/workflows/ci.yml`
**Triggers**: Push to `main`, all PRs

Three parallel jobs:

### lint
- Checkout + install tools (uv, bun)
- Backend: `ruff check .` + `ruff format --check .`
- Frontend: `bun run lint`

### backend-test
- Service containers: Postgres 16 (port 5432) + Mailpit (ports 1025/8025)
- Install: `uv sync --frozen`
- Run: `uv run pytest -v`
- Environment matches docker-compose values (DATABASE_URL, JWT_SECRET, FERNET_KEY, SMTP on localhost)

### frontend-test
- Install: `bun install --frozen-lockfile`
- Run: `bun run test`

## App Platform Deployment

**File**: `.do/app.yaml`

Single app with 3 components:

### backend (Web Service)
- Dockerfile: `backend/Dockerfile`
- HTTP port 8000, health check at `/health`
- Routes: `/api` prefix
- Env vars: DATABASE_URL (from DB binding), JWT_SECRET, FERNET_KEY, EMAIL_PROVIDER=resend, RESEND_API_KEY, CORS_ORIGINS, MAGIC_LINK_BASE_URL

### frontend (Web Service)
- Dockerfile: `frontend/Dockerfile`
- HTTP port 3000, health check at `/`
- Routes: `/` catch-all
- Build arg: BACKEND_URL set to backend's private URL

### db (Managed Postgres)
- Engine PG, version 16
- Size: db-s-1vcpu-1gb ($15/mo)
- Dedicated database name: `nelson` (cluster shared with other apps)
- DATABASE_URL auto-injected into backend

## Key Decisions

- **Release command**: `cd backend && uv run alembic upgrade head` — runs migrations before each deploy
- **Dedicated DB**: `nelson` database within managed Postgres cluster (allows sharing cluster across apps)
- **Routing**: App Platform routes `/api/*` to backend, everything else to frontend. Frontend Next.js rewrites proxy `/api` to backend internally (same as local dev).
- **Custom domain**: Added via DO dashboard post-deploy (CNAME to app URL)
- **Auto-deploy**: From `main` branch on push

## No Application Code Changes

All config is already env-driven. No changes to backend or frontend code needed.

## Testing

- CI pipeline verified by pushing a PR and seeing green checks
- Post-deploy: health check returns 200, magic link auth flow works with Resend email
